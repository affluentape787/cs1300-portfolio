Candy Shop React Interface

React Aggregating + Filtering + Sorting

Part 0: Project Motivation and Overview

In this project, I explore the development process of an interactive interface in React by using interface components, tying the components to an internal data state, and implementing aggregator, filter, and sort functions. This is my first experience with React, and I decided to develop a rather simple website that represents a candy shop where users can choose snacks that they want, as well as filter by type and size of snack and sort by price (ascending or descending). I love snacks so this was extremely fun to make!

Part 1: Overall Goal and Value of Application

The goal of my application is to represent an online snack shop where users can sift through a collection of snacks and add and remove snacks from their cart. The cart keeps track of the number of each snack as well as a price total. The user may be craving a specific snack type (i.e. chips, candy, cookies) and can filter by the type of snack they want. They may also be looking for a specific snack size (i.e. individual, family, party) depending on the purpose for purchasing the snacks, so they can also filter by size as well. They may want to sort the snacks from least to most expensive or most to least expensive so there are two sort options for those capabilities as well.

Considering Usability Principles

  1. The filters and sort capabilities should be placed at the top and centered on the page so the user can easily see and select the filter/sort that they want.
  2. The filters should use checkboxes while the sort should use circular radio buttons since multiple filters can be applied but only one sort function can be used at a time, matching the user's mental model.
  3. The 'Add to Cart' and 'Remove from Cart' are different colors, and more specifically, the 'Remove from Cart' button is outlined in red to match the user's mental model that red means stop/remove.
  4. The cart, which is purposely located toward the top of the screen, reminds users what snacks are actually inside, including the quantity and price of each snack as well as the total cart price. Users should not have to scroll down to find the price of a specific snack in their cart, mentally keep track of the quantity of each snack, or manually calculate how much they need to pay.

Part 2: Components

Snack Component

The Snack component contains important details about each snack, including an image, name, type of snack, size, price, add to cart button, and remove from cart button. These are laid out in Bootstrap grid fashion on the page of the application.

Part 3: User-Triggered State Changes

Filtering and Sorting

How Filtering Works

When the user selects/de-selects a certain filter such as one of the snack type filters, the filter is added to/removed from an array state variable called 'typeFilters' that contains all the snack type filters that are selected (via a setter function). Then, based on the 'typeFilters' array, the state variable called 'type' may change. If no type filters are now selected, then 'type' is set to "all" and all snack types are shown; if one type filter is selected, then 'type' is set to that specific snack type and only snacks of that type are shown; if more than one type filter is selected, type is set to "none" and no snacks are shown since each snack is only of one type. The same process is used for the snack size filter.

How Sorting Works

I also use a state variable for the 'sortType' (price ascending or price descending), which utilizes the setter function 'setSortType.' When the user selects a type of sort, the 'sortType' is altered through 'setSortType,' and the appropriate sorting function is used on the (filtered) 'snackData.'
